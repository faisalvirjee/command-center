First-Time Project Setup
Do this once when creating a new project
▶
1
Create the project — Click + New in the topbar. Enter the project name, company (e.g. Asgardian Holdings LLC), and property address. All 48 standard tasks are pre-loaded automatically.
2
Fill in project details — Go to Settings → Project. Set the General Contractor, Start Date, and Estimated Completion. Set Report Period (e.g. Q1 2026) and Report Date for investor reports.
3
Set phase budgets — Go to Finance tab. Enter land cost, construction budget, builder management fee, and contingency. These feed into the Phase Budget Tracker and investor report automatically.
4
Add investors — Go to Investors tab. Add each investor's name, email, capital amount, and equity %. The app will calculate projected returns once you enter the estimated sale price in Finance.
5
Connect Google Drive — Go to Settings → Drive. Click Connect Drive and approve Google's permission screen. Your data will sync to cpm_data.json in your Drive and be available across all browsers.
6
Create the Field Ops Sheet — Go to Settings → Google Sheets Integration. Click 📊 Create Field Ops Sheet. CPM creates a pre-formatted Google Sheet in your shared Drive folder and auto-fills the Sheet ID.
7
Share the Sheet with Ahmed — Open the newly created Sheet in Google Drive. Click Share and add Ahmed's Google account with Editor access. See the Field Ops Sheet section below for what to tell him.
8
Set up the Publish App — Go to Settings → Publish App. Click Set Password and create a viewer password for Ahmed. Hit Publish Read-Only Snapshot to push the first snapshot to Drive.
💡 The project won't appear in the topbar dropdown if its status is set to Archived. If a project seems to have disappeared, go to Settings and change the status back to Active.

Field Ops Sheet — Setup & Rules
Creating the sheet and what Ahmed fills in
▶
What "Create Field Ops Sheet" does

One button creates a Google Sheet in your shared Drive folder with all tasks pre-loaded, columns sized, header frozen, color-coded, and columns A & B protected with an edit warning. The Sheet ID is automatically pasted into CPM Settings — no manual copy/paste needed.

The 11 columns
A — Phase
Pre-filled by CPM. Ahmed does not edit.
B — Task Name
Pre-filled by CPM. Ahmed does not edit.
C — Planned Start
Ahmed fills: when does he plan to start this task?
D — Planned End
Ahmed fills: when does he plan to finish?
E — Actual Start
Ahmed fills: when did work actually start?
F — Actual End
Ahmed fills: when did work actually finish?
G — % Complete
0 to 100. No % symbol needed.
H — Status
Not Started / In Progress / Complete / Blocked
I — Cost ($)
Numbers only. No $ sign. (e.g. 4200.00)
J — Vendor
Contractor or supplier name for this task.
K — Blocker / Notes
Any issues, delays, or notes. Blockers appear on the CPM dashboard.
Rules for Ahmed
⚠ Columns A and B are protected — if Ahmed sees a warning when clicking those cells, tell him to click Cancel and leave them alone.
✓
Fill in columns C through K as work progresses. Update each row in place — don't add new rows.
✓
Dates in MM/DD/YYYY format (e.g. 06/01/2026). The app handles the conversion automatically.
✓
Rows marked N/A are cost-only tasks (like GC Management Fee). Leave their date and % columns blank.
✗
Do NOT add, delete, or rearrange columns. Adding a column between existing ones shifts everything and breaks the sync.
✗
Do NOT add rows above the header row or merge cells in the data range.
💡 Ahmed CAN add his own notes columns to the right (column L and beyond). The sync only reads A through K — anything beyond that is ignored safely.
Recommended Sheet formatting (do this after creating)

The sheet is already formatted with frozen headers, color-coded columns, and column widths. If you want to add borders or alternating row colors for Ahmed's readability, right-click any cell in Google Sheets and use Format → Alternating colors. This won't affect the sync at all.

Regular Sync Workflow
Pulling Ahmed's updates into CPM
▶
1
Ahmed fills in or updates task rows in the Google Sheet throughout the week.
2
When you're ready to pull his updates, open CPM → Settings → Google Sheets Integration → click 🔄 Sync from Sheet.
3
CPM reads the current sheet and opens a Staging Review modal showing exactly what changed. Nothing is committed yet.
4
Review each row. Use the checkboxes to select what to bring in. Click Approve & Commit when ready.
What the staging badges mean
🟢 NewTask exists in the sheet but not yet in CPM — will be added as a new task.
🟡 ChangedTask exists in both — Ahmed updated one or more fields. Shows exactly what changed (before → after).
🔴 DuplicateSame Phase + Task Name appears more than once in the sheet. Skipped — fix in the sheet.
⚠ Not in SheetTask exists in CPM but wasn't found in the sheet. Informational only — no action taken, never deleted.
💡 Recommended cadence: sync once a week, or after each major milestone (e.g. phase completion, significant cost entry). The staging view always shows you exactly what's about to change before anything is committed.
⚠ The sync only pulls data — it never pushes changes back to the Sheet or modifies Ahmed's rows. Your CPM data is always the system of record.

Adding & Removing Tasks
Managing the task list as the project evolves
▶
Adding a new task
1
Go to Phases & Tasks → expand the phase → click + Add Task to Phase X.
2
Enter the task name. The task is created in CPM immediately.
3
Go to Settings → Google Sheets Integration → click 📤 Push New Tasks to Sheet.
4
CPM reads the current sheet, finds which tasks are missing, and appends them at the bottom. Ahmed sees the new row next time he opens the Sheet.
💡 Always add tasks in CPM first, then push. Ahmed should never add rows to the Sheet directly.
Removing a task
1
In Phases & Tasks, click the ✏ pencil on the task row to open the edit modal.
2
Click Delete Task at the bottom left and confirm.
3
The task is removed from CPM. Ahmed's sheet still has the row — it shows as ⚠ Not in Sheet on the next sync (informational only, no data is harmed).
⚠ If Ahmed has already filled in data for a deleted task, that data is not synced back — it stays in his sheet only. Decide whether to delete before Ahmed has entered data for a task, where possible.
Marking a task as cost-only (no schedule)
1
Open the task edit modal (✏ pencil).
2
Check the No schedule (cost-only task) checkbox. The date fields collapse.
3
The task will show NO SCH badge in the task table and export N/A in the date columns so Ahmed knows to skip them.
💡 The GC Management Fee task is cost-only by default. Other tasks like legal fees or permit costs that don't have a schedule duration work well as cost-only tasks.
Editing a task's details

Click the ✏ pencil on any task row. You can update: task name, status, % complete, cost, vendor, planned dates, actual dates, blocker/note, and the no-schedule flag. Click Save Task when done. The change saves locally immediately and syncs to Drive within 1.5 seconds.

Publishing to Ahmed (Read-Only Viewer)
Giving Ahmed a live read-only dashboard
▶
How it works

When you publish, CPM encrypts the full project data with AES-256 and writes it to your shared Google Drive folder. Ahmed opens a standalone viewer app on GitHub, enters the password, and sees a read-only dashboard with all phases, tasks, financials, and investor details. He can never edit anything through the viewer.

First-time setup
1
Go to Settings → Publish App.
2
Check that the Shared Drive Folder ID matches the folder you want to publish to. Click Change if needed — paste the folder ID from the Drive URL.
3
Click Set Password and create a viewer password. Give this password to Ahmed (share it securely — not in the same message as the link).
4
Send Ahmed this URL: https://faisalvirjee.github.io/command-center/cpm-viewer.html — he bookmarks it once and uses it forever.
Publishing an update
1
Click 📤 Publish Read-Only Snapshot in Settings → Publish App.
2
The modal walks you through a print step (for the PDF) and then writes the encrypted file to Drive.
3
Last Published timestamp updates — Ahmed now sees the new data when he opens the viewer.
💡 Publish whenever you want Ahmed to see fresh data — after a sync, after a ledger update, or before a site meeting. The viewer always shows whatever you last published.
⚠ Ahmed sees everything — all investor names, amounts, and equity splits. This is intentional since he is both the GC and an investor. Do not publish sensitive information you don't want him to see.

Investor Report
Generating and exporting the monthly report
▶
Before you generate
1
Set the Report Period in Settings (e.g. Q2 2026) — this appears on the report cover and header strip.
2
Set the Report Date in Settings — this is the "As Of" date on the cover. Leave blank to use today's date.
3
Make sure Investor Capital is entered in the Investors tab and Phase Budgets are set in Finance. These drive the financial tables in the report.
Writing the narrative sections

Go to the Reports tab. The report renders live on screen. You'll see an editable text area for the Executive Summary — type your letter to partners there. The "Next Steps" and "Investment Terms" fields are edited in the Project Info area of Settings.

Exporting to PDF
1
Click 🖨 Print / Export PDF in the topbar (or on the Reports tab).
2
In the modal, select which sections to include. Investor Report + Dashboard are checked by default.
3
Click Print Selected. Your browser's print dialog opens — choose Save as PDF as the destination.
4
The filename is auto-set to CompanyName_ProjectName_Period_InvestorReport — rename if needed before saving.
💡 Chrome and Edge give the cleanest PDF output. In Chrome: Print → Save as PDF → More settings → Scale: 85% for best fit.

Troubleshooting
Common issues and how to fix them
▶
Project not showing in dropdown

The project status is probably set to Archived. Go to Settings → Project Status and set it back to Active. Archived projects are hidden from the selector to keep things clean.

Drive won't connect

Reload the page and try again. If it still fails, clear your browser cache (Ctrl+Shift+Delete) and try a fresh load. If the issue persists, make sure you're using Chrome or Edge — the Google APIs work best in those browsers.

Sync fails with "Sheets access denied" (403)

Your Drive session expired or the Sheets permission wasn't granted. Go to Settings → Drive → Connect Drive and reconnect. On the Google approval screen, make sure to approve all requested permissions.

Sheet ID not recognized / Sheet not found (404)

Re-paste the full Google Sheet URL into the Google Sheet URL or ID field in Settings. The app extracts the ID automatically from the URL. Make sure the Sheet is in your Google Drive (not a shared-with-me file you don't own).

Task shows ⚠ Not in Sheet after sync

This task exists in CPM but wasn't found in Ahmed's sheet. It's informational — nothing was deleted. Use 📤 Push New Tasks to Sheet to add it to the sheet, or check with Ahmed if the task name got edited.

Column data looks shifted / wrong field in wrong place

Ahmed likely inserted or deleted a column. The sync reads columns A through K positionally — if any column was moved, everything shifts. Have Ahmed restore the original column order, or recreate the sheet using Create Field Ops Sheet and re-share.

"Invalid Date" showing in task table

This was a known issue with dates from earlier versions that stored them in MM/DD/YYYY format instead of YYYY-MM-DD. Fixed in v2.1.1+. To clear any remaining ones: open the affected task's edit modal (✏), clear the date fields and re-enter them, then save.

Forgot password

On the login screen, click Forgot Password?. This connects to your Google Drive (OAuth) to verify your identity and lets you set a new password. No Drive connection = no password reset, so keep Drive connected.

Data seems to have reset or reverted

Data loads from Drive when you connect, overwriting what's in the browser. If you made changes without Drive connected and then connected, Drive data wins. To prevent: always connect Drive before making changes. The ☁ Save button in the topbar forces an immediate Drive save.

💡 When in doubt: click ☁ Save, then reload the page and reconnect Drive. This forces a clean sync from the canonical Drive file and usually resolves data inconsistencies.
